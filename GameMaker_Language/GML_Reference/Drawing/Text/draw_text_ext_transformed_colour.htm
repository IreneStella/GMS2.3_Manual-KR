draw_text_ext_transformed_colour

이 함수는 draw_text()함수를 기본으로 draw_text_ext(), draw_text_transformed() 그리고 draw_text_colour()을 합친 함수입니다. 이 함수는 모든 문자 그리기 함수 중 가장 기본이 되는 함수이며 그리는 점에서 가장 부드러운 함수입니다. 중요한건 인자의"w"은 크기를 1을 기준으로 둡니다, 그래서 만약 크기가 1이 아니라면, 크기값이 바뀐만큼의 비율로 변경됩니다 . 예를들어, 2의 크기로 설정하고 300픽셀마다 줄바꿈을 하라고 인자에 넣으면, 문자는 예상과 다르게 주어진 픽셀을 넘길 것입니다. 대신 크기에 맞춰 150픽셀로 설정하면 잘 작동됩니다.

NOTE: HTML5 타겟일때 WebGL 이 불가능하면 변화 블렌딩(Gradient blending)은 불가능 합니다,비록 블렌드 색을 설정할수 있고 처음에 주어진 색으로 폰트가 그려질수 있지만. 이 방법으로 그려지는 블렌딩은 복사된후 캐시에 저장됩니다 그리고나서 필요하면 사용됩니다, 이는 최적화와 거리가 멀며 만약 당신이 다양한 색을 사용하면 게임은 느려집니다. 그러므로 당신은 font_set_cache_size() 을 사용하여 캐시의 크기를 제한을 걸면 괜찮아집니다.

 

문법:

draw_text_ext_transformed_colour(x, y, string, sep, w, xscale, yscale, angle, c1, c2, c3, c4, alpha);

인자 설명
x 그려질 문자열의 x좌표.
y 그려질 문자열의 y좌표.
string 그려질 문자열
sep 각 문자열줄들간의 픽셀간격.
w 줄바꿈의 기준인 최대가로픽셀값.
xscale 수평 크기.
yscale 수직 크기.
angle 문자의 각도.
c1 그려질 문자열 좌측상단의 색.
c2 그려질 문자열 우측상단의 색.
c3 그려질 문자열 우측하단의 색
c4 그려질 문자열 좌측하단의 색.
alpha 문자열의 투명도.

 

Returns:

N/A

 

Example:

draw_set_halign(fa_center);
draw_set_valign(fa_middle);
draw_text_ext_transformed_colour(200, 200, "My name is" + global.name +".", 20, 200, 0.5, 0.5, 180, c_blue, c_blue, c_navy, c_navy, 1);

이 위의 코드는 파란색에서 검은 파란색으로 변화되면서 그려지고, 룸에서 (200, 200)위치에 그려집니다, 또한 위 아래가 뒤집혀서 그려지고 폰트의 반정도 크기만큼 그려집니다. 200픽셀만큼 줄바꿈이 되며 각 줄은 20픽셀 만큼의 빈 공간을 가집니다.
