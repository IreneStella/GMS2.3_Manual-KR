draw_text_colour

이 함수는 draw_text()와 비슷하지만 문자의 색과 알파값(투명도)를 정할수 있습니다, 그리고 이는 기존에 설정된 색과 알파값(투명도) 를 사용하지 않고 인자의 설정을 따릅니다.

NOTE: HTML5 타겟일때 WebGL 이 불가능하면 변화 블렌딩(Gradient blending)은 불가능 합니다,비록 블렌드 색을 설정할수 있고 처음에 주어진 색으로 폰트가 그려질수 있지만. 이 방법으로 그려지는 블렌딩은 복사된후 캐시에 저장됩니다 그리고나서 필요하면 사용됩니다, 이는 최적화와 거리가 멀며 만약 당신이 다양한 색을 사용하면 게임은 느려집니다. 그러므로 당신은 afont_set_cache_size() 을 사용하여 캐시의 크기를 제한을 걸면 괜찮아집니다.

 

문법:

draw_text_colour(x, y, string, c1, c2, c3, c4, alpha);

인자 설명
x 그려질 x의 좌표.
y 그려질 y의 좌표.
string 그려질 문자열.
c1 그려질 문자열 좌측상단의 색.
c2 그려질 문자열 우측상단의 색.
c3 그려질 문자열 우측하단의 색
c4 그려질 문자열 좌측하단의 색.
alpha 문자열의 투명도.

 

반환값:

N/A

 

예시:

draw_set_colour(c_white);
draw_text(100, 100, "Health");
draw_text_colour(100, 200, string(health), c_lime, c_lime, c_green, c_green, 1);

이위의 코드는 같은줄에 두개의 문자열이 그려집니다, 첫번째 문자열은 하얀색이며 (기본값에 의해서) 두번째 문자열은 라임이 초록색으로 변화되며 그려집니다.
